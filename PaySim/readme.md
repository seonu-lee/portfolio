
## **PaySim 금융 거래 데이터 분석 & 사기 탐지 프로젝트**

### 프로젝트 개요

* **목표**: 시뮬레이션 금융 거래 데이터를 활용하여 사용자 행동 패턴 분석 및 사기 거래 탐지
* **데이터**: PaySim (실제 금융 거래 로그 기반 시뮬레이션 데이터)
* **도구**: Python (Pandas, NumPy, Scikit-learn, Matplotlib/Seaborn)

PaySim은 실제 모바일 금융 서비스의 거래 흐름을 모사한 데이터로,
사용자 행동 분석과 사기 탐지 모델 연구에 널리 활용된다.

---

### 분석 과정

1. **EDA (Exploratory Data Analysis)**

   * 거래 유형(CASH_IN, CASH_OUT, TRANSFER 등) 분포 분석
   * 정상 거래 vs 사기 거래 패턴 비교
   * 거래 금액, 빈도, 시간(step) 기반 이상 패턴 탐색

2. **RFM 분석**

   * Recency, Frequency, Monetary 지표 정의
   * 사용자 금융 활동 수준에 따른 고객 가치 분류
   * 고빈도·고액 거래 사용자와 저활동 사용자 비교

3. **AARRR 퍼널 분석**

   * Acquisition → Activation → Revenue → Retention → Referral 단계 정의
   * 거래 단계별 사용자 이탈 지점 파악
   * 사기 거래가 퍼널에 미치는 영향 분석

4. **유동성 및 PATHWAY 분석**

   * 사용자 간 자금 흐름(Network Pathway) 시각화
   * 반복적 송금 경로 및 비정상 자금 순환 탐지
   * 사기 계정 중심의 거래 연결 구조 분석

5. **세그멘테이션 및 프로파일링**

   * 거래 행동 기반 사용자 군집화(KMeans 등)
   * 각 세그먼트별 거래 특성 및 리스크 프로파일 정의
   * 정상 사용자 vs 고위험 사용자 행동 차이 비교

6. **사기 탐지 분석**

   * 사기 여부(label) 기준 클래스 불균형 문제 확인
   * 거래 금액, 빈도, 전이 패턴 기반 특징 설계
   * 사기 거래의 주요 패턴 및 발생 조건 도출

---

### 결과

* 사기 거래는 **TRANSFER / CASH_OUT 유형**에 집중되는 경향 확인
* 정상 사용자와 사기 계정 간 **거래 경로(PATHWAY) 구조 차이** 발견
* RFM 및 군집 분석을 통해 **고위험 사용자 세그먼트 식별**
* 단일 지표보다 **행동 흐름 + 네트워크 관점 분석**이 사기 탐지에 효과적임을 확인

---

### 배운 점

* 금융 데이터 분석에서 **EDA와 도메인 이해의 중요성** 체감
* 단순 분류 성능보다 **사기 발생 메커니즘 해석**이 더 중요함을 인식
* 퍼널, 세그멘테이션, 네트워크 분석을 결합한 **입체적 사용자 분석 경험**
* 향후 지도학습 기반 사기 탐지 모델(RandomForest, XGBoost 등) 확장 가능성 확인
